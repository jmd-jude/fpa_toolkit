# Box AI Enrichment — Migration Design

**Context:** The current AI enrichment feature calls the Anthropic API directly from `python/enrich.py`. This works well but routes PHI through Anthropic's infrastructure, which requires a separate HIPAA BAA. FPAmed is on Box **Enterprise Advanced** (confirmed on invoices), which includes Box AI API access with BAA coverage built in. Migrating enrichment to use Box AI keeps all file content within the Box trust boundary — the app only ever receives extracted key-value pairs, never file bytes. Railway deployment stays as-is.

*Research sourced from Box developer documentation via `https://developer.box.com/llms.txt`.*

## Recommended Endpoint: `POST /2.0/ai/extract_structured`

The `/2.0/ai/ask` endpoint (what you'd reach for first) is designed for Q&A and returns freeform text. A better fit is `POST /2.0/ai/extract_structured`, which:

- Takes a **predefined schema** of fields to extract (exactly what we want: date + description)
- Returns **validated key-value pairs** in the field types you specify (`date`, `string`)
- Explicitly supports **PDF, TIFF, PNG, JPEG** file formats
- Has **native OCR for scanned PDFs** — no text layer required
- Accepts a `metadata_template` or inline `fields` array

This replaces the "prompt for JSON, parse the response" pattern in the current implementation with a schema-driven call that returns structured data directly.

Freeform alternative: `POST /2.0/ai/extract` accepts a single file + a prompt (plain text, JSON, or XML format describing what to extract). More flexible, same single-file-per-request limit.

## Scanned PDF Support

This was the main caveat in the original draft. **The concern is largely resolved:**

The Enhanced Extract Agent (Gemini 2.5 Pro) has native multimodal OCR. Per Box documentation it handles:
- Scanned documents and photos
- Handwritten notations
- Watermarks and distinctive fonts
- Intricate / complex layouts

The "text layers on scanned PDFs" feature previously mentioned by Box's product team is a separate convenience feature (making scans text-searchable in Box UI). It is not a prerequisite for Box AI extraction — the vision models handle scanned content directly.

**Implication:** The main technical reason to keep the current Anthropic-direct approach (rendering pages as images ourselves for vision inference) goes away. Box AI handles the scanned PDF case natively with the right model.

## Model Selection

Full model control is available via the `ai_agent` parameter on all AI endpoints. You specify the model by exact API name. Relevant options:

| Model API name | Notes |
|---|---|
| `google__gemini_2_5_pro` | Default for Enhanced Extract Agent; 1M token context, strongest OCR/vision |
| `google__gemini_2_5_flash` | Faster, still strong; default for Standard Extract |
| `aws__claude_4_5_haiku` | Lightweight, cost-efficient — analogous to current Anthropic direct setup |
| `aws__claude_4_5_sonnet` | Higher quality, 200K token input |
| `openai__gpt_5_mini` | Optimized for well-defined tasks |

Note the double-underscore separator (`provider__model_name`). These are Box's routing identifiers, not Anthropic/Google/OpenAI model IDs directly.

For date + description extraction from medical PDFs (often scanned), **`google__gemini_2_5_pro` is the recommended starting point** — it was purpose-built for complex document extraction and has the strongest scanned doc handling.

## API Request Shape

```python
# POST /2.0/ai/extract_structured

{
  "items": [{"type": "file", "id": "1234567890"}],  # one file per request
  "fields": [
    {
      "key": "document_date",
      "type": "date",
      "description": "The document's own date — from letterhead, header, footer, or signature block. Not a received or filed stamp.",
      "prompt": "What is the date this document was created or signed?"
    },
    {
      "key": "description",
      "type": "string",
      "description": "One sentence (15 words or fewer) describing what this document is.",
      "prompt": "Briefly describe what type of document this is."
    }
  ],
  "ai_agent": {
    "type": "ai_agent_extract_structured",
    "long_text": {
      "model": "google__gemini_2_5_pro"
    }
  }
}
```

Response shape:
```json
{
  "document_date": "2023-04-15",
  "description": "Hospital discharge summary following inpatient psychiatric evaluation."
}
```

## Rate Limits

- **General Box API limit:** 1,000 requests/minute per user
- **Box AI-specific limits:** Not documented — empirical testing needed
- **Retry guidance:** Box returns HTTP 429 with a `retry-after` header; use exponential backoff
- **Safe starting point:** 5 concurrent workers (same as current `enrich.py`), monitor for 429s

For 100 files at 5 workers, worst case is ~20 sequential batches. Even at 3 seconds per call that's ~60 seconds — within acceptable range for an async job.

## Revised `enrich.py` Sketch

Compared to the current implementation this removes: pymupdf rendering, base64 encoding, Anthropic SDK, and `ANTHROPIC_API_KEY`. The Box access token already in use is sufficient.

```python
"""
Box AI Document Enricher — uses Box AI extract_structured instead of Anthropic direct.
Files never leave Box custody; BAA covered by Enterprise Plus.

Usage:
  .venv/bin/python3 python/enrich.py \
    --manifest-file /tmp/test/slug_manifest.csv \
    --token <box_access_token> \
    [--model google__gemini_2_5_pro] \
    [--workers 5]
"""

import argparse
import csv
import json
import threading
import concurrent.futures
import requests

BOX_EXTRACT_URL = "https://api.box.com/2.0/ai/extract_structured"
ENRICHABLE_EXTENSIONS = {".pdf"}

FIELDS = [
    {
        "key": "document_date",
        "type": "date",
        "description": (
            "The document's own date — from letterhead, header, footer, or signature block. "
            "Not a received or filed stamp."
        ),
        "prompt": "What date was this document created or signed?",
    },
    {
        "key": "description",
        "type": "string",
        "description": "One sentence (15 words or fewer) describing what type of document this is.",
        "prompt": "Briefly describe what type of document this is in one short sentence.",
    },
]


def call_box_ai(token: str, file_id: str, model: str) -> dict:
    headers = {
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json",
    }
    payload = {
        "items": [{"type": "file", "id": file_id}],
        "fields": FIELDS,
        "ai_agent": {
            "type": "ai_agent_extract_structured",
            "long_text": {"model": model},
        },
    }
    response = requests.post(BOX_EXTRACT_URL, headers=headers, json=payload, timeout=30)
    response.raise_for_status()
    data = response.json()
    return {
        "ai_date": data.get("document_date") or "",
        "ai_description": data.get("description") or "",
    }


def enrich_row(token: str, model: str, row: dict) -> dict:
    try:
        return call_box_ai(token, row["File ID"], model)
    except Exception as e:
        print(f"  Warning: enrichment failed for {row['Name']}: {e}", flush=True)
        return {"ai_date": "", "ai_description": ""}


def main():
    parser = argparse.ArgumentParser(description="Box AI Document Enricher")
    parser.add_argument("--manifest-file", required=True)
    parser.add_argument("--token", required=True)
    parser.add_argument("--model", default="google__gemini_2_5_pro")
    parser.add_argument("--workers", type=int, default=5)
    args = parser.parse_args()

    with open(args.manifest_file, newline="", encoding="utf-8") as f:
        rows = list(csv.DictReader(f))

    pdf_rows = [r for r in rows if r.get("Extension", "").lower() in ENRICHABLE_EXTENSIONS]
    total = len(pdf_rows)
    print(f"Box AI enrichment: {total} files to process (model: {args.model})", flush=True)

    row_by_id = {r["File ID"]: r for r in rows}
    counter = [0]
    lock = threading.Lock()

    def process(row):
        result = enrich_row(args.token, args.model, row)
        with lock:
            counter[0] += 1
            n = counter[0]
        print(f"  [{n}/{total}] {row['Name']} → {result['ai_date'] or 'no date'}", flush=True)
        return row["File ID"], result

    with concurrent.futures.ThreadPoolExecutor(max_workers=args.workers) as executor:
        futures = {executor.submit(process, r): r for r in pdf_rows}
        for future in concurrent.futures.as_completed(futures):
            file_id, result = future.result()
            row_by_id[file_id]["AI Date"] = result["ai_date"]
            row_by_id[file_id]["AI Description"] = result["ai_description"]

    for row in rows:
        row.setdefault("AI Date", "")
        row.setdefault("AI Description", "")

    fieldnames = list(rows[0].keys())
    with open(args.manifest_file, "w", newline="", encoding="utf-8") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()
        writer.writerows(rows)

    print(f"Enrichment complete → {args.manifest_file}", flush=True)


if __name__ == "__main__":
    main()
```

## What Changes vs. Current Implementation

| | Current (Anthropic direct) | Box AI migration |
|---|---|---|
| Auth | `ANTHROPIC_API_KEY` env var | Box access token (already in use) |
| File handling | Download → pymupdf render → base64 encode images | File ID only — Box handles everything |
| Scanned PDFs | Works (vision model sees rendered page images) | Works (Gemini 2.5 Pro native OCR/vision) |
| HIPAA coverage | Requires separate Anthropic BAA | Covered by existing Enterprise Plus BAA |
| Cost model | Per-token (~$0.05–0.10 / 100 files w/ Haiku) | Included in Enterprise Plus (AI Units) |
| Model control | Full (`claude-haiku-4-5`, `claude-sonnet-4-5`, etc.) | Full via `ai_agent` parameter (see model table above) |
| Dependencies | `anthropic`, `pymupdf` (rendering) | `requests` only (already available) |
| Railway changes | None | None — add `BOX_AI_MODEL` env var optionally |
